# Online Electronics Store

## Опис проєкту

Online Electronics Store — це повнофункціональний backend-застосунок, розроблений на базі Spring Boot.

Система реалізує функціональність маркетплейсу, де користувачі можуть:

- реєструватися та авторизуватися
- додавати власні товари
- переглядати товари інших продавців
- додавати товари до кошика
- оформлювати замовлення
- спілкуватися з продавцем через чат
- адміністратор може керувати категоріями товарів

Проєкт побудований із використанням чистої багатошарової архітектури та production-підходів до безпеки.

---

# Архітектура

Застосунок реалізований за принципом **Layered Architecture (багатошарова архітектура)**:

Controller Layer Service Layer Repository Layer Domain Model

Додатково:

- DTO-шар ізолює API-контракти
- Mapper-шар відповідає за перетворення Entity ↔ DTO
- Security-шар реалізує аутентифікацію та авторизацію
- Глобальний обробник помилок забезпечує єдину структуру відповіді

---

# Структура пакетів

config        → Конфігурація Spring controller    → REST та MVC контролери dto           → Data Transfer Objects exception     → Глобальні та кастомні винятки mapper        → Логіка перетворення DTO ↔ Entity model         → JPA-сутності та enum repository    → Spring Data репозиторії security      → JWT та компоненти безпеки service       → Бізнес-логіка

# Ключові інженерні рішення

## 1. Розділення Cart і Order

- `Cart` — поточний стан кошика користувача.
- `CustomerOrder` — знімок даних у момент оформлення.

Це дозволяє зберігати історичну коректність замовлень навіть при зміні ціни товару.

---

## 2. Стратегія знімка (Snapshot)

У `CustomerOrder` зберігаються:

- ім’я клієнта
- телефон
- адреса
- `priceAtOrder` (у OrderItem)

Таким чином історія замовлення залишається незмінною.

---

## 3. Чат прив’язаний до замовлення

Chat ↔ CustomerOrder

Кожен чат стосується конкретного замовлення, що спрощує комунікацію між покупцем і продавцем.

---

## 4. Lazy Loading

Використовується:

FetchType.LAZY

Це покращує продуктивність і запобігає надлишковому завантаженню даних.

---

## 5. orphanRemoval

При видаленні товару з кошика відповідний `CartItem` автоматично видаляється з бази.

Це підтримує цілісність даних.

---

## 6. Використання Enum

Enum застосовуються для:

- Role (USER / ADMIN)
- OrderStatus
- Gender

Це підвищує типобезпечність і запобігає некоректним станам.

---

## 7. Динамічний розрахунок суми кошика

@Transient getTotal()

Сума кошика не зберігається в базі, а обчислюється динамічно.

# Архітектура безпеки

Реалізована подвійна модель безпеки:

## 1. REST API — JWT (Stateless)

- Аутентифікація через JWT
- SessionCreationPolicy.STATELESS
- JwtAuthFilter
- JwtAuthEntryPoint
- BCrypt хешування паролів
- Role-based авторизація

Шлях:

/api/**

---

## 2. Web-рівень — Session-based

- Form login
- CSRF-захист
- Обмеження доступу для ADMIN

Це дозволяє ізолювати REST API від MVC-частини застосунку.
